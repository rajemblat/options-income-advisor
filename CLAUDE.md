# Workflow

- After finishing each significant block of work (a feature, a bugfix, a meaningful
  refactor), commit the change and push it to `origin main` automatically — do not wait
  to be asked. Write a clear commit message per the standard git commit conventions.
- Skip this for trivial/exploratory changes (e.g. answering a question, one-off debugging
  that gets reverted) — only commit+push work that should actually land on `main`.
- Still pause and ask first if a change is destructive, ambiguous in scope, or touches
  something the user hasn't clearly asked for.
